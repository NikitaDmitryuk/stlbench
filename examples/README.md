# stlbench Examples

## Gendalf (3-part model)

The `gendalf/main/` directory contains a three-part model of Gandalf stored via Git LFS.
This example demonstrates all five stlbench commands using an ELEGOO Mars 5 Ultra config.

All commands below assume you are in the **repository root** (next to `stlbench/` and `examples/`).

---

### 1. `info` -- Inspect parts before doing anything

```bash
stlbench info -i examples/gendalf/main -c configs/mars5_ultra.toml
```

Shows a table for each STL file:
- AABB dimensions (X, Y, Z)
- Volume, vertex and face counts
- Whether the part fits the printer bed at 1:1
- Maximum scale factor (`s_max`)
- How many copies would fit with `fill`

Use this to understand what you are working with before running any other command.

---

### 2. `scale` -- Uniform scaling to fit the build volume

```bash
stlbench scale \
  -i examples/gendalf/main \
  -o examples/gendalf/scaled \
  -c configs/mars5_ultra.toml
```

Computes a single scale factor so that every part fits inside the printer volume.
Writes scaled STL files to `examples/gendalf/scaled/`.

Useful options:
- `--dry-run` -- show the scale factor and report without writing files
- `--no-upscale` -- never enlarge parts (cap scale at 1.0)
- `--method conservative` -- use the most pessimistic axis mapping

---

### 3. `layout` -- Pack scaled parts onto print plates

```bash
stlbench layout \
  -i examples/gendalf/scaled \
  -o examples/gendalf/plates \
  -c configs/mars5_ultra.toml
```

Takes already-scaled STL files and arranges them onto rectangular print plates
using `rectpack`. Exports `plate_01.stl` + `plate_01.json`, etc.

Useful options:
- `--dry-run` -- report plate count without exporting
- `--gap-mm 1.0` -- override gap between parts

---

### 4. `autopack` -- Scale + pack all parts onto one plate

```bash
stlbench autopack \
  -i examples/gendalf/main \
  -o examples/gendalf/autopack \
  -c configs/mars5_ultra.toml
```

Finds the maximum scale factor at which **all** parts fit onto a single print plate
simultaneously. Exports `autopack_plate.stl` + `autopack_plate.json`.

This is the "one-command" workflow: no separate scale + layout steps.

---

### 5. `fill` -- Maximum copies of a single part

```bash
stlbench fill \
  -i examples/gendalf/main/sword.stl \
  -o examples/gendalf/fill_sword \
  -c configs/mars5_ultra.toml
```

Packs as many copies of a single STL as possible onto one plate.
Exports `fill_plate.stl` + `fill_plate.json` with positions and copy count.

Add `--scale` to first scale the part down to fit the bed:

```bash
stlbench fill \
  -i examples/gendalf/main/sword.stl \
  -o examples/gendalf/fill_sword_scale \
  -c configs/mars5_ultra.toml \
  --scale
```

---

### Output files

| Command    | Output directory                | Key files                                  |
|------------|---------------------------------|--------------------------------------------|
| `info`     | *(none -- console only)*        |                                            |
| `scale`    | `examples/gendalf/scaled/`      | `*.stl` (one per input part)               |
| `layout`   | `examples/gendalf/plates/`      | `plate_01.stl`, `plate_01.json`, ...       |
| `autopack` | `examples/gendalf/autopack/`    | `autopack_plate.stl`, `autopack_plate.json`|
| `fill`     | `examples/gendalf/fill_sword/`   | `fill_plate.stl`, `fill_plate.json`        |

All output directories are listed in `.gitignore` and are not committed.

---

### Notes

- The Gendalf model files are tracked with **Git LFS**. Run `git lfs pull` if the STL
  files appear as pointer files.
- Supports are **not** generated by stlbench. After exporting, open the STL in
  Lychee, Chitubox, or another slicer to add supports.
